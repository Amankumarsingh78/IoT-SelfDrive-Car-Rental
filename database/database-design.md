# Database Design

## Project Name

IoT-Enabled Self-Drive Car Rental Platform

---

## Database Name

car_rental

---

## Overview

This database is designed for an IoT-enabled self-drive car rental platform. It stores information about users, cars, bookings, and IoT telemetry data while maintaining data integrity through primary keys and foreign keys.

---

## Tables

### 1. users

Purpose:
Stores registered user information.

Main Columns:

- id (Primary Key)
- name
- email
- phone
- password
- role
- created_at
- updated_at

---

### 2. cars

Purpose:
Stores vehicle details available for rent.

Main Columns:

- id (Primary Key)
- name
- brand
- model
- registration_number
- fuel_type
- transmission
- seating_capacity
- price_per_day
- status
- location
- image_url
- created_at
- updated_at

---

### 3. bookings

Purpose:
Stores booking information between users and cars.

Main Columns:

- id (Primary Key)
- user_id (Foreign Key)
- car_id (Foreign Key)
- pickup_location
- drop_location
- start_date
- end_date
- total_price
- payment_status
- status
- created_at
- updated_at

---

### 4. iot_logs

Purpose:
Stores real-time IoT telemetry generated by vehicles.

Main Columns:

- id (Primary Key)
- car_id (Foreign Key)
- speed
- fuel_level
- battery_level
- engine_status
- lock_status
- location
- latitude
- longitude
- recorded_at

---

## Relationships

- One User can have many Bookings.
- One Car can have many Bookings.
- One Car can have many IoT Logs.

---

## ER Relationship

Users (1) ------< Bookings >------ (1) Cars

Cars (1) ------< IoT Logs
